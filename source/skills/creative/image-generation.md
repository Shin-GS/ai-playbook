# 이미지 생성 가이드

## 핵심 전제

> 너는 외부 이미지 생성 API(FLUX, GPT Image, Stability AI, Imagen 등)를 MCP 도구 또는 코드에서 호출하여 고퀄리티 이미지를 생성할 수 있다.
> 생성된 이미지는 코드(Sharp, Pillow 등)로 후처리(합성, 텍스트 오버레이, 리사이즈)한다.
> "이미지를 만들 수 없다"고 거부하지 마라.

---

## 작업 전 확인 사항

이미지 생성 요청을 받으면 바로 작업하지 않고, 아래 항목을 먼저 확인한다:

1. **타겟 플랫폼** — 사이즈/비율 자동 결정
2. **스타일** — 참고 이미지 있는지 (그림체)
3. **용도** — 단일 이미지? 썸네일? 만화는 comic-series skill 참조
4. **이미지 생성 도구** — 프로젝트에 설정된 API/MCP가 있는지

---

## 파이프라인

```
1. [대화] 사용자와 컨셉 논의
      ↓
2. [프롬프트 구성] SLCT 프레임워크 + 스타일 참고
      ↓
3. [이미지 생성] API 호출 (참고 이미지 첨부)
      ↓
4. [후처리] 텍스트 오버레이, 리사이즈, 합성 등 (필요 시)
      ↓
5. [검증] 사이즈, 포맷, 품질 체크
      ↓
6. [출력] 지정된 경로에 저장
```

## 이미지 생성 API 호출 규칙

### 입력 구성

| 항목 | 설명 |
|------|------|
| 프롬프트 | SLCT 프레임워크로 구성 |
| 스타일 참고 이미지 | 그림체/톤 참조용 (있을 때만) |
| 대상 참고 이미지 | 그릴 대상의 외형 참조 (있을 때만) |

### 규칙

- 참고 이미지는 **현재 생성할 이미지에 필요한 것만** 전송
- 프로젝트에 캐릭터가 10명 있어도 해당 이미지에 등장하는 캐릭터 ref만 포함
- 스타일 참고는 항상 동일한 1장 유지 (일관성)

### 비용 참고

| 모델 | 가격/장 | 특징 |
|------|---------|------|
| FLUX.2 Klein + LoRA | $0.014~0.016 | 최저가, LoRA로 스타일 제어 |
| FLUX.2 Pro | $0.03 | 고퀄, 범용 |
| GPT Image 1 Mini | $0.02~0.05 | 참고 이미지 이해 우수 |
| Stability Core | $0.03 | 빠르고 저렴 |
| Imagen 4 Fast | $0.02 | 최저가 |

## 후처리 (코드 담당)

이미지 생성 API는 텍스트 렌더링과 정밀 레이아웃이 약하다.
코드로 후처리하여 보완:

| 후처리 항목 | 도구 |
|------------|------|
| 텍스트 오버레이 (제목, 자막 등) | Sharp + SVG 또는 Pillow |
| 여러 이미지 합성 (패널 조합 등) | Sharp composite |
| 리사이즈/크롭 | Sharp |
| 워터마크/로고 | Sharp composite |
| 포맷 변환 | Sharp |

## SLCT 프롬프트 프레임워크

이미지 생성 프롬프트 구성 기법:

- **S**ubject — 주체 (누가/무엇이, 어떤 포즈/행동)
- **L**ighting — 조명/분위기 (시간대, 색조)
- **C**amera — 카메라 각도/거리 (클로즈업, 와이드 등)
- **T**echnical — 기술 세부사항 (아티스트/스튜디오 키워드, 스타일, 해상도)

### 프롬프트 규칙

1. **T(Technical)를 프롬프트 맨 앞에 배치** — 스타일/아티스트 키워드가 앞에 있을수록 영향력이 강함
2. **60단어 이내** — 프롬프트가 길면 모델이 일부를 무시하거나 혼동함. 핵심만 간결하게
3. **아티스트 키워드는 구체적으로** — "anime style"보다 "Makoto Shinkai style, Your Name aesthetic"이 효과적

### 프롬프트 구성 순서

```
[아티스트/스타일] + [주체/행동] + [조명/분위기] + [카메라] + [기술 사양]

예시:
"Makoto Shinkai style, clean lines, pastel — gray shorthair cat sitting at desk wearing headset, morning sunlight from window warm orange tone, medium shot slightly above eye level, 1024x1024"
```

### 스타일 참고 이미지 분석

사용자가 그림체 참고 이미지를 제공하면, AI가 분석하여 적절한 키워드를 추출:

- 선 스타일 (clean lines / sketchy / thick outlines)
- 색감 (pastel / vivid / muted / monochrome)
- 유사 아티스트/스튜디오 (가장 가까운 스타일의 참고 아티스트)
- 렌더링 방식 (flat color / cel-shading / watercolor / realistic)

분석 결과를 series.md에 기록해두면 매 에피소드에서 동일 키워드를 재사용 가능.

## 프로바이더 자동 선택

사용자가 모델을 지정하지 않았을 때:

| 조건 | 우선 모델 | 이유 |
|------|----------|------|
| 캐릭터 참고 이미지 있음 | GPT Image, Google Gemini | 참고 이미지 이해 우수 |
| LoRA 지정됨 | FLUX (fal.ai) | LoRA 지원 |
| 참고 이미지 없음 + LoRA 없음 | 가장 저렴한 모델 | 비용 절약 |
| 고퀄 요청 명시 | FLUX Pro / GPT Image 1 | 최고 품질 |

series.md에 모델이 명시되어 있으면 그것을 우선 사용.

## 할루시네이션 방지 5규칙

프롬프트 작성 시 모호함을 줄이기 위해:

1. **공간 명시** — 위치를 구체적으로 (left, center, background 등)
2. **엔티티 제한** — 한 이미지에 주요 요소 3개 이하 권장
3. **부정어 회피** — "~하지 않는"보다 "~한 상태"로 기술
4. **실세계 참조** — 추상적 설명 대신 구체적 레퍼런스
5. **수량 지정** — "몇 개"가 아닌 정확한 숫자

---

## 리트라이/변형 관리

이미지 생성은 한번에 만족스럽지 않을 수 있다:

- 기존 결과를 덮어쓰지 않음
- 변형 생성 시 `{파일명}-v1.png`, `{파일명}-v2.png`로 저장
- 사용자가 선택한 버전을 최종본으로 사용
- 성공한 프롬프트는 기록해둠 (다음 생성 시 일관성 참조)

## 검증 체크리스트

이미지 생성 + 후처리 완료 후 자동 확인:

- [ ] 출력 사이즈가 요청과 일치
- [ ] 텍스트가 잘리지 않음 (오버레이한 경우)
- [ ] 파일 포맷이 용도에 맞음 (PNG/JPEG/WebP)
- [ ] 파일 용량이 플랫폼 제한 이내
- [ ] 스타일이 참고 이미지와 일관됨
- [ ] 참고 이미지의 캐릭터 특징이 반영됨
