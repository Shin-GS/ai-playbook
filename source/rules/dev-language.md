# 개발 언어 및 소통 규칙

## 1. 언어 규칙

### 에이전트 응답
- 에이전트 응답은 **한국어**로 작성한다.
- 설명, 질문, 제안 등 모든 자연어 소통은 한국어.

### 코드
- 코드(변수명, 함수명, 클래스명, 메서드명, API 경로)는 모두 **영어**로 작성한다.
- 로그 메시지는 영어로 작성한다.
- 코드 주석은 한국어 허용 (개발자 간 소통 목적).

## 2. 정확성 원칙

- 확실하지 않은 내용은 추측하지 않고 **모른다고 명확히 답변**한다.
- 불확실한 부분은 관련 코드나 문서를 직접 확인한 후 답변한다.
- "아마", "~일 수 있음" 같은 모호한 표현 대신 확인된 사실만 전달한다.

## 3. 변경 범위 제한

- **요청받은 부분만** 수정한다.
- 명시적으로 언급되지 않은 코드는 변경하지 않는다.
- 관련 코드 개선이 보이더라도 요청 범위를 벗어나면 별도로 제안만 한다.
- "ついでに(겸사겸사)" 수정 금지 — 의도하지 않은 부작용 방지.

## 4. 되묻기 원칙 (상세)

상세 규칙은 `ask-before-assume` 자산을 참조하라.

요약:
- 해석이 2가지 이상 가능하면 → 되물어라
- 삭제/대체/의존성 추가 → 되물어라
- 기존 패턴을 따르는 명확한 구현 → 자율 진행
